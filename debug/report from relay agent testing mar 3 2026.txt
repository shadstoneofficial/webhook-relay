Subject: Action Required: Implement Webhook Dispatch Logic in PowerLobster Backend

To: PowerLobster Backend Team

Body:

Hi Team,

The Relay Server infrastructure is live and verified. Agents are successfully connecting and waiting for events. However, no events are being sent from the main application when triggers fire (e.g., new DM, new task).

The Missing Piece: You need to implement the actual HTTP POST call to the Relay Server whenever a relevant event occurs in your system.

Implementation Guide:

When an event happens (e.g., on_dm_created ), you must:

1. Check: Does the recipient user/agent have a webhook_url saved in their profile?
2. Filter: Does the user have the dm.received trigger enabled?
3. Dispatch: Send a POST request to their webhook_url .
Request Format:

- Method: POST
- URL: The value stored in agent.webhook_url (e.g., https://relay.powerlobster.com/api/v1/webhook/agt_... )
- Headers:
  - Content-Type: application/json
  - x-powerlobster-signature : sha256=<HMAC_SHA256(timestamp + "." + json_body, WEBHOOK_SECRET)>
  - x-powerlobster-timestamp : <current_unix_timestamp>
- Body:
  ```
  {
    "event": "dm.received",
    "data": {
      "message_id": "...",
      "content": "..."
    }
  }
  ```
Note: The WEBHOOK_SECRET must match the one configured in the Relay Server environment variables.

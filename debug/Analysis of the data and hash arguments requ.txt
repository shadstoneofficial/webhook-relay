###  Analysis of the "data and hash arguments required" Error
Klyve (the agent) is reporting a very specific error: data and hash arguments required . This is a classic Bcrypt error message.

Where does it happen? It happens inside bcrypt.compare(data, hash) .

- data is the plain text password (the API key sent by the agent).
- hash is the hashed password (stored in the DB).
The Error Means: One of the arguments passed to bcrypt.compare is undefined or null .

Let's trace the code path in packages/relay-server/src/websocket/manager.ts :

```
          // Verify credentials
          const agent = await getAgentByRelayId(message.relay_id);
          if (!agent || !await verifyApiKey(message.api_key, agent.
          api_key_hash)) {
            return this.sendError(connection, 
            'invalid_credentials', 'Invalid relay_id or API key');
          }
```
And packages/relay-server/src/services/auth.ts :

```
export async function verifyApiKey(apiKey: string, hash: string): 
Promise<boolean> {
  return bcrypt.compare(apiKey, hash);
}
```
Scenario 1: agent is null If getAgentByRelayId returns null (agent not found), the code !agent is true, so the || short-circuits. verifyApiKey is NEVER called. So, if the agent didn't exist, we would get "Invalid relay_id or API key", NOT "data and hash arguments required".

Scenario 2: message.api_key is missing If the agent sends a JSON without api_key (or uses a different key name like relay_api_key ), then message.api_key is undefined .

- verifyApiKey(undefined, hash) is called.
- bcrypt.compare(undefined, hash) throws "data and hash arguments required".
